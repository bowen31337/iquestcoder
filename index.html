The LoopCoder Architecture

To address deployment constraints without sacrificing capacity, the IQuest-Coder-V1-Loop variant introduces a recurrent mechanism. Instead of a standard linear pass, transformer blocks are executed in two fixed iterations, effectively doubling the logical depth.

1

Iteration 1: Position-Shifted Processing

Input embeddings are processed through transformer layers to build an initial hidden state representation.

2

Iteration 2: Gated Attention Mix

The model computes both Global Attention (attending to Iteration 1's KV pairs) and Local Attention (causal). A learned gate combines these to refine logical coherence.

Multilingual Synergy

Contrary to traditional scaling laws, mixing syntactically-related programming languages provides a "Synergy Gain." For example, training on Java significantly improves C# performance due to high transfer coefficients.

The "Thinking" Path

Training with explicit reasoning traces (Thinking Models) triggers an emergent ability for autonomous error-recovery. Standard Instruct models often fail to self-correct in long-horizon tasks, while Thinking models can backtrack and revise.
